Hello, I'm Kakyo, and i'm a factorio addict :D

It always bother me that several calculators out there didn't happen to include the calculations for modules usage - which is a big part of the game.

So i take the torch myself and start this little app.



There are still a few big features to support (like mod recipes) but i believe that as a first version this is good enough and I can start to receive some feedback from this.

Github Repository

The project can be found in this repository, please report bug/issues and request features over there

Known issues

  • Currently, whenever an item can be produced by more then one recipe, the first one is always selected. Manually addition is required - no way to add as child.
  • "Add children" (little tree) button can lead to infinite loop with any recipe that has oil products as requirement.
    E.g.: Empty oil barrel can be created by Full Oil Barrel that can be created by Empty Oil Barrel that can be created by Full Oil Barrel........

Feature Wishlist

  • Custom recipe support

    This system will include all mods out there, the goal here is to allow even personal modifications to mods.

  • Surplus/Shortage

    Display input surplus/shortage and use this value to calculate upstream output. E.g. A sulfur plant working at 80% will cause the sulfuric acid to work at 80%.

  • Ratio view

    Have a different way of seeing the data. Ratio will display recipes only once, regardless of how many others uses it as input, and then sum the production for them all.

  • Save/Share Setup

    A way to easily distribute information and/or discuss ratios.

  • Language support

    I, currently, already read the game files for language (so, technically, all recipes are translated already), but the text on the site itself is not supported and thus i've locked language changes.
